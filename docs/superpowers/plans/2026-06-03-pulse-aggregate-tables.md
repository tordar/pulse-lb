# Pulse-LB Aggregate Tables Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace 8 heavy aggregating queries on the pulse-lb stats page (and the songs/artists/albums list pages) with indexed reads from seven per-user aggregate tables, rebuilt atomically at end-of-sync-chain.

**Architecture:** Add seven derived tables (`agg_alltime`, `agg_year`, `agg_hour`, `agg_day`, `agg_song`, `agg_artist`, `agg_album`) plus a `sync_state.last_aggregated_at` column. A new `rebuildAll(username)` orchestrator fires at end-of-sync inside `app/api/sync/[username]/route.ts` when the chain terminates (`completed && added===0`) and `lastListenedAt > lastAggregatedAt`. All read paths in `lib/db/queries/stats.ts` and `lib/db/queries/topItems.ts` are rewritten to read from the aggregates, keeping the same TypeScript signatures so callers don't change.

**Tech Stack:** Next.js 16 App Router on Vercel · Drizzle ORM 0.45 · `@neondatabase/serverless` 1.1 HTTP driver · Postgres (Neon).

## Notes for the implementer

### Drizzle / Neon constraints

- **The HTTP driver does not support `db.transaction()`.** Drizzle's `db.transaction()` requires a persistent connection. For multi-statement atomic execution, import `neon` from `@neondatabase/serverless` directly and use `sqlClient.transaction([q1, q2, …])` — that bundles statements into one HTTP request with BEGIN/COMMIT on the Neon side.
- All existing queries already use `withRetry()` from `lib/db/retry.ts`. New code that talks to the DB MUST also wrap calls in `withRetry()` for parity.
- `neon-http` returns query results as `{ rows: T[] }`. All existing code casts via `as unknown as { rows: T[] }`. Follow the same pattern.

### Existing patterns to follow

- One-off scripts live under `scripts/` (e.g. `_check_rgid.ts`, `backfill-release-groups.ts`). They `import "dotenv/config"`, instantiate their own neon client, and exit on completion. The bootstrap and equivalence scripts in this plan follow that pattern.
- Migrations: change `lib/db/schema.ts`, run `npm run db:generate`, then `npm run db:migrate`. Drizzle names migrations automatically (`000N_<random_two_words>.sql`).
- No test framework is installed. We verify correctness via the equivalence script (Task 4) which compares old and new query output row-by-row against the dev DB.

### Common SQL building blocks

`group_key` per entity type (computed at write time, stored in aggregate row):
- Song: `COALESCE(l.recording_mbid::text, '~' || l.track_name)`
- Album: `l.release_name || '|' || COALESCE(l.artist_name, '')` (the `WHERE release_name IS NOT NULL` filter excludes the awkward case)
- Artist: just `l.artist_name` (already a unique key in the existing query)

`effective_ms`: `COALESCE(SUM(COALESCE(l.duration_ms, r.length_ms)), 0)::bigint`

`recording_mbid` representative: `mode() WITHIN GROUP (ORDER BY l.recording_mbid) FILTER (WHERE l.recording_mbid IS NOT NULL)`

## Task 2: Implement `rebuildAll` orchestrator

**Files:**
- Create: `lib/db/aggregates/rebuild.ts`

This file owns the SQL that rebuilds each aggregate table for one user. Eight exported functions: seven `rebuildX` per table, and one `rebuildAll` that runs them all as a single Neon HTTP transaction.

- [ ] **Step 1: Create the file with the orchestrator skeleton and `rebuildAlltime`**

Create `lib/db/aggregates/rebuild.ts`:

```ts
import { neon, type NeonQueryFunction } from "@neondatabase/serverless";
import { withRetry } from "@/lib/db/retry";

// We use the raw neon client (not drizzle) for two reasons:
//   1. drizzle-orm/neon-http does not support db.transaction().
//   2. neon's HTTP transaction([...]) API bundles multiple statements
//      into one HTTP round trip with BEGIN/COMMIT. That's how we get
//      atomicity for the multi-table rebuild.
const sql = neon(process.env.DATABASE_URL!) as NeonQueryFunction<false, false>;

export async function rebuildAll(username: string): Promise<void> {
  await withRetry(() =>
    sql.transaction([
      sql`DELETE FROM agg_alltime WHERE user_name = ${username}`,
      sql`DELETE FROM agg_year    WHERE user_name = ${username}`,
      sql`DELETE FROM agg_hour    WHERE user_name = ${username}`,
      sql`DELETE FROM agg_day     WHERE user_name = ${username}`,
      sql`DELETE FROM agg_song    WHERE user_name = ${username}`,
      sql`DELETE FROM agg_artist  WHERE user_name = ${username}`,
      sql`DELETE FROM agg_album   WHERE user_name = ${username}`,

      buildAlltime(username),
      buildYear(username),
      buildHour(username),
      buildDay(username),
      buildSong(username),
      buildArtist(username),
      buildAlbum(username),
    ]),
  );
}

function buildAlltime(username: string) {
  return sql`
    INSERT INTO agg_alltime (
      user_name, total_plays, effective_ms,
      distinct_artists, distinct_albums, distinct_songs,
      first_played, last_played, duration_coverage_pct
    )
    SELECT
      ${username}::text,
      COUNT(*)::int,
      COALESCE(SUM(COALESCE(l.duration_ms, r.length_ms)), 0)::bigint,
      COUNT(DISTINCT l.artist_name)::int,
      COUNT(DISTINCT l.release_name)::int,
      COUNT(DISTINCT COALESCE(l.recording_mbid::text, l.track_name))::int,
      MIN(l.listened_at),
      MAX(l.listened_at),
      ROUND(
        100.0 * COUNT(*) FILTER (
          WHERE l.duration_ms IS NOT NULL OR r.length_ms IS NOT NULL
        ) / NULLIF(COUNT(*), 0),
        1
      )::float8
    FROM listens l
    LEFT JOIN recordings r ON r.mbid = l.recording_mbid
    WHERE l.user_name = ${username}
    HAVING COUNT(*) > 0
  `;
}
```

The `HAVING COUNT(*) > 0` ensures we do NOT insert a row for a user with zero listens. The page handles a missing `agg_alltime` row as the empty state.

function buildYear(username: string) {
  return sql`
    INSERT INTO agg_year (user_name, year, plays, hours)
    SELECT
      ${username}::text,
      EXTRACT(YEAR FROM l.listened_at)::int,
      COUNT(*)::int,
      ROUND(
        COALESCE(SUM(COALESCE(l.duration_ms, r.length_ms)), 0) / 1000.0 / 3600,
        2
      )::float8
    FROM listens l
    LEFT JOIN recordings r ON r.mbid = l.recording_mbid
    WHERE l.user_name = ${username}
    GROUP BY EXTRACT(YEAR FROM l.listened_at)::int
  `;
}

function buildHour(username: string) {
  return sql`
    INSERT INTO agg_hour (user_name, hour, plays)
    SELECT
      ${username}::text,
      h.hour,
      COALESCE(c.plays, 0)::int
    FROM generate_series(0, 23) AS h(hour)
    LEFT JOIN (
      SELECT EXTRACT(HOUR FROM listened_at)::int AS hour, COUNT(*)::int AS plays
      FROM listens
      WHERE user_name = ${username}
      GROUP BY hour
    ) c USING (hour)
  `;
}

function buildDay(username: string) {
  return sql`
    INSERT INTO agg_day (user_name, date, plays)
    SELECT
      ${username}::text,
      listened_at::date,
      COUNT(*)::int
    FROM listens
    WHERE user_name = ${username}
    GROUP BY listened_at::date
  `;
}
```

Note `buildHour` always writes 24 rows even when some hours are zero. This matches the existing read query's `generate_series` behavior.

`buildDay` writes only days with at least one play. The read query (`dailyListeningByYear`) fills zeros via `generate_series` on the year range.

function buildSong(username: string) {
  // UNION ALL of year-scoped rows (scope = year) and all-time rows (scope = 0 (all-time sentinel)).
  // Same SELECT shape; the only differences are the scope expression and the
  // GROUP BY columns.
  return sql`
    INSERT INTO agg_song (
      user_name, scope, group_key, track_name, artist_name,
      plays, effective_ms, caa_id, caa_release_mbid, recording_mbid
    )
    -- Year-scoped
    SELECT
      ${username}::text,
      EXTRACT(YEAR FROM l.listened_at)::int,
      COALESCE(l.recording_mbid::text, '~' || l.track_name),
      (array_agg(l.track_name ORDER BY l.listened_at DESC))[1],
      l.artist_name,
      COUNT(*)::int,
      COALESCE(SUM(COALESCE(l.duration_ms, r.length_ms)), 0)::bigint,
      (array_agg(l.caa_id ORDER BY l.listened_at DESC)
        FILTER (WHERE l.caa_id IS NOT NULL))[1],
      (array_agg(l.caa_release_mbid ORDER BY l.listened_at DESC)
        FILTER (WHERE l.caa_release_mbid IS NOT NULL))[1],
      mode() WITHIN GROUP (ORDER BY l.recording_mbid)
        FILTER (WHERE l.recording_mbid IS NOT NULL)
    FROM listens l
    LEFT JOIN recordings r ON r.mbid = l.recording_mbid
    WHERE l.user_name = ${username}
    GROUP BY
      EXTRACT(YEAR FROM l.listened_at)::int,
      COALESCE(l.recording_mbid::text, '~' || l.track_name),
      l.artist_name

    UNION ALL

    -- All-time (scope = 0)
    SELECT
      ${username}::text,
      0::int,
      COALESCE(l.recording_mbid::text, '~' || l.track_name),
      (array_agg(l.track_name ORDER BY l.listened_at DESC))[1],
      l.artist_name,
      COUNT(*)::int,
      COALESCE(SUM(COALESCE(l.duration_ms, r.length_ms)), 0)::bigint,
      (array_agg(l.caa_id ORDER BY l.listened_at DESC)
        FILTER (WHERE l.caa_id IS NOT NULL))[1],
      (array_agg(l.caa_release_mbid ORDER BY l.listened_at DESC)
        FILTER (WHERE l.caa_release_mbid IS NOT NULL))[1],
      mode() WITHIN GROUP (ORDER BY l.recording_mbid)
        FILTER (WHERE l.recording_mbid IS NOT NULL)
    FROM listens l
    LEFT JOIN recordings r ON r.mbid = l.recording_mbid
    WHERE l.user_name = ${username}
    GROUP BY
      COALESCE(l.recording_mbid::text, '~' || l.track_name),
      l.artist_name
  `;
}

import "dotenv/config";
import { neon } from "@neondatabase/serverless";
import { rebuildAll } from "../lib/db/aggregates/rebuild";

async function main() {
  const sql = neon(process.env.DATABASE_URL!);
  const rows = (await sql`
    SELECT DISTINCT user_name FROM listens ORDER BY user_name
  `) as Array<{ user_name: string }>;

  if (rows.length === 0) {
    console.log("No users in listens table. Nothing to bootstrap.");
    return;
  }

  console.log(`Bootstrapping aggregates for ${rows.length} user(s).`);
  for (const { user_name } of rows) {
    const t0 = Date.now();
    await rebuildAll(user_name);
    await sql`
      INSERT INTO sync_state (user_name, last_aggregated_at)
      VALUES (${user_name}, now())
      ON CONFLICT (user_name) DO UPDATE SET last_aggregated_at = now()
    `;
    console.log(`  ${user_name} — ${Date.now() - t0}ms`);
  }
  console.log("Done.");
}

main().catch((e) => {
  console.error(e);
  process.exit(1);
});
```
